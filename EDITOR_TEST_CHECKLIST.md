# 编辑器中文输入测试清单

## 测试环境准备
1. 启动开发服务器：`cd apps/web && pnpm dev`
2. 访问编辑器页面：`http://localhost:5173/editor/test-work`
3. 确保使用中文输入法（拼音输入法）

## 手动测试场景

### ✅ 场景1：基础拼音输入
**测试步骤：**
1. 点击编辑器空白区域
2. 输入拼音 `ni hao shi jie`
3. 在候选词出现时观察编辑器内容
4. 选择候选词 `你好世界`

**预期结果：**
- 候选词阶段：编辑器中不应出现拼音字母 `ni hao shi jie`
- 选择后：正确显示 `你好世界`
- 字数统计：显示 4

### ✅ 场景2：句子中间插入中文
**测试步骤：**
1. 编辑器中输入英文：`Hello world!`
2. 将光标放在 `Hello` 和 `world` 之间
3. 输入拼音 `ni hao`，选择 `你好`

**预期结果：**
- 结果应为：`Hello 你好 world!`
- 光标位置在 `你好` 后面
- 字数统计正确

### ✅ 场景3：选中替换中文
**测试步骤：**
1. 编辑器中输入：`今天天气很好`
2. 选中 `天气` 两个字
3. 输入拼音 `tian qi`，选择 `天气`

**预期结果：**
- 选中内容被正确替换
- 结果仍为：`今天天气很好`
- 没有多余字符

### ✅ 场景4：快速连续输入
**测试步骤：**
1. 快速连续输入多个中文词组
2. 如：`今天天气很好我想出去玩`
3. 不停顿地快速输入

**预期结果：**
- 不丢字、不乱序
- 每个字都能正确输入
- 字数统计准确

### ✅ 场景5：混合输入（中英文混输）
**测试步骤：**
1. 输入：`我爱 React 和 Vue`
2. 在中文和英文之间切换输入法

**预期结果：**
- 中英文都能正确显示
- 没有编码问题
- 字数统计：中文按字算，英文按单词算

### ✅ 场景6：标点符号输入
**测试步骤：**
1. 输入中文句子：`你好，世界！`
2. 使用中文标点输入法

**预期结果：**
- 中文标点正确显示
- 不会变成英文标点

## 自动化测试准备

编辑器内容区域已添加 `data-testid="editor-content"`，方便后续编写自动化测试：

```typescript
// 例如在 Playwright 或 Cypress 中
await page.locator('[data-testid="editor-content"]').fill('测试内容');
const content = await page.locator('[data-testid="editor-content"]').textContent();
```

## 常见问题排查

1. **如果出现拼音字母**：检查 `compositionstart` 事件处理
2. **如果光标位置错乱**：检查 `compositionend` 事件处理
3. **如果字数统计不准**：检查 StatusBar 的字数计算逻辑
